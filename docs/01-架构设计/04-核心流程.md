# 04 · 核心流程

> 五条流程覆盖系统全部运行时行为；每条标注决策来源与失败路径（14.1 Fail Fast）。

## 流程一 · 任务完工链（submit → diff 核验 → Validator → commit → Done）

```text
Worker: POST submit {lease_token}
   │
   ▼
[1] fencing 检查：token 匹配且未过期？ ──否──► 409 LEASE_LOST（幽灵提交拒收，D-05①）
   │是
   ▼
[2] git status --porcelain （目标项目树）
    按 write_scope 前缀分拣：
      in_scope  = 声明范围内的变更
      out_scope = 范围外的变更 ∩ 非其他 Running 任务的写范围
   │
   ├─ out_scope ≠ ∅ ──► [F] scope-diff 失败（D-10）
   │                     注意：范围外但属于其他 Running 任务写范围的文件 → 忽略
   │                     （共享树上的邻居在制品，README 偏差 #3）
   ▼
[3] 注册 Validator 依次执行（FR-08.2）
    v1 内置：scope-diff（上一步）、cmd（任意命令，如 cargo check）、
             artifact-exists（声明产物齐全）
   │
   ├─ 任一失败 ──► [F]
   ▼
[4] git add <in_scope> && git commit
    -m "task(nd_xxx): {title} [attempt {n}]"
    → 得 commit_hash
   │
   ▼
[5] 同一 PG 事务：
    attempt.outcome='done' + artifact 插入 + task_state='done'
    + 直接下游 ready 重算 + event task.done + NOTIFY
   │
   ▼
   200 {verdict: done}   ← Done 必有可读产物（14.1 可恢复性3：先 commit 后置 Done）

[F] 失败路径（D-06/D-07）：
    attempt.outcome='validation_failed', failure=报告
    ├─ attempt_seq < max_attempts ──► task_state='ready'，failure 注入下次输入（原 Task 重开）
    └─ 达上限 ──► task_state='failed' + event task.failed
                  （人工介入或 Planner 经 replan-context 重规划，D-07 两级策略）
```

**[4]→[5] 的缝**：commit 成功但 PG 写失败 → 重试 submit 时检测「write_scope 无 diff 但 HEAD 含该任务 commit」→ 幂等复用该 commit 完成 [5]（14.1 一致性 2）。

## 流程二 · 租约与接力（D-05 四条协议的运行时展开）

```text
                     ┌────────────────────────────────────────┐
                     │  回收器（tokio interval，每 15s）         │
                     │  UPDATE ... WHERE lease_expires < now() │
                     │  → task_state='ready', owner=NULL,      │
                     │    lease_token=NULL（①旧执行作废）        │
                     │  → attempt.outcome='lease_expired'      │
                     │  → event task.lease_expired             │
                     │  ※ 达 max_attempts 则直接 'failed'       │
                     └────────────────────────────────────────┘
Agent-A claim ──► 干活 ──► [挂起/崩溃] ··· 租约过期，如上回收
                                              │
Agent-B claim-next ──► 命中该任务（attempt_seq=2）
   │  响应含 handover 块：
   │    previous_attempts（上一手记录）
   │    work_in_progress（git status ∩ 该任务写范围 → 在制品清单）②
   │    progress_snapshot（graph_version + 上游产物版本）④
   ▼
Agent-B 审查现场 ──► POST handover-review（decision: reuse/discard/partial）③
   │                    ├─ discard → server 执行 git checkout -- <discarded_paths>
   │                    └─ 报告入 attempt.handover + event（审计，不得隐瞒）
   ▼
开工（未 review 就 submit → 409 HANDOVER_REVIEW_REQUIRED）
   │
   ▼ （此后与流程一相同）

迟到的 Agent-A：任何回带旧 lease_token 的调用 → 409 LEASE_LOST（fencing）
```

## 流程三 · Worker 主循环（写进 Worker Agent 提示词的行为契约）

```text
loop {
    r = POST /v1/tasks/claim-next {capabilities}
    match r {
      204 + hint.expandable_packages 非空
          → 通知 Planner 会话下钻（或人工触发 expand）    // FR-05.6
      204 + 无提示 → sleep(poll_interval) / 等 SSE graph.published
      200 → {
          if handover 存在: 审查在制品 → POST handover-review
          后台心跳: 每 lease_ttl/3 POST heartbeat
          按 spec 干活（只写 write_scope 内路径）
          POST submit
            ├─ 200 done   → continue
            └─ 422 reopened → 本轮结束（失败上下文已注入，下次 claim 可能又是自己）
      }
    }
}
```

## 流程四 · Planner 展开与 Contract 变更

### 4a Expand（D-01 全自动）

```text
触发（三选一）：人工 CLI `dag plan expand <pkg>` / Worker 204 hint / Planner 自主巡查
   │
   ▼
Planner 会话读子图（GET /graph）→ 按 08/09/10 方法论拆分 → POST /graph/publish
   │
   ▼
server 端编译校验（dag-core，全部 FR-03.3 规则）：
   环 / 悬空依赖 / 重复 ID / 类型不变量（package 有 children、task 无）/
   兄弟 Ready 任务写范围前缀相交 / 缺失输入 / Validator 未注册
   ├─ 违规 → 422 全部违规项（FR-03.4，一次给全）
   ▼
原子发布：graph_version++ → 新节点入库 → ready 重算 → event graph.published → NOTIFY
   │
   ▼
空闲 Worker 经 SSE/轮询立即认领（D-01：无人工门，发布即可执行）
```

### 4b Contract 版本（D-08 semver）

```text
Contract 产物完工（流程一）→ Planner/Worker 声明 bump 级别 → POST /contracts/publish
   ├─ patch → 直接生效，无标记
   ├─ minor → 生效 + 通知性事件（下游不强制返工）
   └─ major → 挂起（contract.pending_major）
              │
              人工: POST approve-major        // 仅人可放行（403 拦 agent）
              │
              ▼
        受影响子图计算（edge 传递闭包 ∩ 消费该 Contract 的节点）
        → 逐节点标记 needs_replan + event contract.impact_marked（FR-10.4）
        → Planner 经 replan-context 生成返工子图（D-07 升级路径复用）
```

## 流程五 · 范围操作（Pause / Close / Drain，D-04）

```text
人工: POST /packages/{id}/scope {action: close, reason, permanent?}
   │（agent 调用 → 403 HUMAN_ONLY）
   ▼
同事务：scope_state='closed' + event package.scope_changed
        + 子树 ready 全部重算为 false（预计算列维护）
   │
   ▼
效果（即时）: 子树任务从 claim-next 消失（Blocked 派生态，D-16）
效果（Drain）: 子树内已 Claimed/Running 的任务照常跑完流程一   // D-04 默认
   │
   ├─ {force: true} 附加：对每个在跑任务执行
   │     租约作废 + task_state='cancelled' + attempt.outcome='cancelled'
   │     + git checkout -- <该任务写范围>（丢弃在制品，D-15）
   ▼
Reopen: scope_state='active' → 子树 ready 重算 → 排队恢复
```

## 失败矩阵（14.1 可恢复性对照）

| 崩的是 | 后果 | 恢复 |
|--------|------|------|
| Worker | 租约过期 → 流程二接力 | 自动 |
| dag-server | 无内存状态（A-01）；SSE 断线用 after_seq 补拉 | 重启即恢复 |
| PG | 系统整体停（唯一真相源） | PG 恢复后一致（事件+投影同事务） |
| submit 中途（commit 后 PG 前） | 幂等重放检测复用 commit | 自动（流程一注） |
| Planner 会话 | 图停留在上一版本，Worker 继续消化存量 Ready | 重开会话，乐观锁防半截发布 |
