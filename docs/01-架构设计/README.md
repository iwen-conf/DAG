## 对需求的收窄与偏差（实现级，均可逆）

| # | 收窄 | 理由 | 回退路径 |
|---|------|------|----------|
| 1 | `write_scope` v1 只支持**路径前缀**（目录/文件），不支持完整 glob | 前缀集合的两两不相交**可判定**（D-10 静态冲突检查因此可靠）；diff 核验退化为 `starts_with` | 引入 globset 后放开，冲突检查改保守近似 |
| 2 | Contract semver **机判延后**：v1 由生产者声明 bump 级别，major 过人工发布门 | schema diff 机判需按 Contract 类型逐个实现（OpenAPI/SQL/Rust trait…），v1 收益低 | D-08 已预留「机判不确定则保守升 major」，v1.1 按类型接入 |
| 3 | Validator 在**共享项目树**上运行，可能看到其他任务的在制品（交叉污染） | v1 单人少量并行，概率低；隔离方案（worktree）已被 D-09=A 否决 | 若实测污染频繁，Validator 加「快照写范围到临时目录」模式 |
| 4 | Task 状态新增 **Cancelled**（`--force` 终止产物） | D-04 的 `--force` 需要终态，不能复用 Failed（语义不同：非任务自身失败） | 已同步回写 [04-状态机](../00-需求分析/04-状态机.md) 待办 |
| 5 | Package 状态**拆列存储**：`scope_state`（人控）+ `plan_state`（Planner 控），Running/Completed 为**派生聚合** | 与 D-16 同一哲学：聚合态落库必然与子树漂移；拆列后两个写者互不覆盖 | 04-状态机 的八态是**展示层投影**，映射表见 02 §2.3 |
