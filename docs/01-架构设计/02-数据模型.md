# 02 · 数据模型（PostgreSQL）

> 原则（D-16/D-17 推广）：**凡是可以从其他真相推出的，不落库**。
> 存储态 = 人或系统的直接决定；派生态 = 查询/视图/预计算列。

## 2.1 存储态 vs 派生态总表

| 信息 | 形态 | 真相源 |
|------|------|--------|
| Task 执行状态（todo→done） | 存储 | `node.task_state` |
| Package 范围状态（active/paused/closed） | 存储 | `node.scope_state`（仅人写，D-18 软约束） |
| Package 规划状态（draft/planning/planned） | 存储 | `node.plan_state`（Planner 写） |
| Package Running/Completed | **派生** | 子树 Task 状态聚合（视图 `v_package_progress`） |
| Blocked | **派生** | 祖先 scope_state + depends 满足度（D-16） |
| Ready | **预计算列** `node.ready`（性能例外，见 §2.6） | 由触达事务同步维护，可全量重算校验 |
| attempts 历史 | 存储 | `attempt` 表（D-05/06/07） |
| 一切变更历史 | 存储 | `event` 表 append-only（D-17） |

## 2.2 DDL

```sql
-- ============ 节点（Package 与 Task 同表，D-02） ============
CREATE TABLE node (
    id              text PRIMARY KEY,              -- 'nd_' + ULID
    graph_version   bigint NOT NULL,               -- 发布该节点的图版本（Planner 乐观锁）
    parent_id       text REFERENCES node(id),      -- 组织树边（FR-02.2：与 depends 分开存）
    path            text NOT NULL,                 -- 物化路径 'root.server.identity.login'
    kind            text NOT NULL CHECK (kind IN ('package','task')),
    layer           text,                          -- D-02 展示标签: project/module/feature…
    role            text,                          -- 10 §10.7: specification/generator/validator/integration
    title           text NOT NULL,
    spec            jsonb NOT NULL DEFAULT '{}',   -- Task 契约全文（目标/输入/验收标准，FR-04）

    -- ---- Task 专用（kind='task'）----
    task_state      text CHECK (task_state IN
                      ('todo','ready','claimed','running','review','done','failed','cancelled')),
    ready           boolean NOT NULL DEFAULT false,  -- 预计算列，见 §2.6
    priority        int NOT NULL DEFAULT 0,          -- D-13
    required_caps   text[] NOT NULL DEFAULT '{}',    -- FR-06.1 能力匹配
    write_scope     text[] NOT NULL DEFAULT '{}',    -- D-10 路径前缀集合（v1 收窄）
    inputs          jsonb NOT NULL DEFAULT '[]',     -- [{artifact_id, version}] FR-04.2
    validators      text[] NOT NULL DEFAULT '{}',    -- FR-08.1 注册名引用
    max_attempts    int NOT NULL DEFAULT 3,          -- D-06: 1 + 2 重试

    -- ---- 租约（D-05）----
    owner           text,                            -- actor 标识
    lease_token     uuid,                            -- fencing：提交必须回带
    lease_expires   timestamptz,

    -- ---- Package 专用（kind='package'）----
    scope_state     text NOT NULL DEFAULT 'active'
                      CHECK (scope_state IN ('active','paused','closed','archived')),
    permanent       boolean NOT NULL DEFAULT false,  -- D-03: closed+permanent = Ignore
    scope_reason    text,                            -- FR-09.5
    scope_actor     text,
    scope_until     timestamptz,
    plan_state      text NOT NULL DEFAULT 'draft'
                      CHECK (plan_state IN ('draft','planning','planned')),

    created_at      timestamptz NOT NULL DEFAULT now(),
    updated_at      timestamptz NOT NULL DEFAULT now(),

    CONSTRAINT task_fields CHECK (kind != 'task' OR task_state IS NOT NULL)
);

-- 执行依赖边（FR-02.2：独立于 parent_id）
CREATE TABLE edge (
    from_id  text NOT NULL REFERENCES node(id),   -- 被依赖者（上游）
    to_id    text NOT NULL REFERENCES node(id),   -- 依赖者（下游）
    PRIMARY KEY (from_id, to_id),
    CHECK (from_id != to_id)
);

-- ============ 事件（D-17，append-only，唯一写入口） ============
CREATE TABLE event (
    id          text PRIMARY KEY,                  -- 'ev_' + ULID
    seq         bigserial UNIQUE,                  -- 全序，投影重放游标
    node_id     text,                              -- 可空：图级事件（publish）无单节点
    actor       text NOT NULL,                     -- 'human' | 'agent:xxx' | 'system:reaper'
    kind        text NOT NULL,                     -- 见 §2.4 事件目录
    payload     jsonb NOT NULL DEFAULT '{}',
    created_at  timestamptz NOT NULL DEFAULT now()
);
-- 附：INSERT 后触发 pg_notify('dag_events', seq::text)，SSE 由此驱动

-- ============ 尝试（D-05/06/07） ============
CREATE TABLE attempt (
    id            text PRIMARY KEY,                -- 'at_' + ULID
    node_id       text NOT NULL REFERENCES node(id),
    seq_no        int  NOT NULL,                   -- 该任务第几次（1 起）
    owner         text NOT NULL,
    started_at    timestamptz NOT NULL DEFAULT now(),
    ended_at      timestamptz,
    outcome       text CHECK (outcome IN
                    ('done','validation_failed','lease_expired','cancelled','error')),
    failure       jsonb,                           -- Validator 输出（FR-08.4，注入下次输入）
    handover      jsonb,                           -- D-05 接力协议：现场审查报告（如实上报）
    UNIQUE (node_id, seq_no)
);

-- ============ 产物（FR-07，D-09=A） ============
CREATE TABLE artifact (
    id           text PRIMARY KEY,                 -- 'ar_' + ULID
    node_id      text NOT NULL REFERENCES node(id),
    attempt_id   text NOT NULL REFERENCES attempt(id),
    paths        text[] NOT NULL,                  -- 写范围内实际变更的文件
    commit_hash  text NOT NULL,                    -- 轻量 Git：每任务一 commit
    digest       text NOT NULL,                    -- 树内容摘要（FR-07.2）
    version      text,                             -- Contract 类产物的 semver（D-08）
    published_at timestamptz NOT NULL DEFAULT now()
);

-- ============ 图版本（D-01 全自动 + D-12 单写者） ============
CREATE TABLE graph_version (
    version     bigint PRIMARY KEY,                -- 单调递增
    planner     text NOT NULL,
    summary     text NOT NULL,                     -- 本次 expand/replan 的说明
    created_at  timestamptz NOT NULL DEFAULT now()
);
```

## 2.3 状态映射：需求八态 → 实现三列

[00-需求分析/04-状态机](../00-需求分析/04-状态机.md) 的 Package 八态是**展示投影**（README 偏差 #5）：

| 需求态 | 实现 |
|--------|------|
| Draft / Planning / Ready | `plan_state = draft / planning / planned` |
| Paused / Closed / Archived | `scope_state = paused / closed / archived` |
| Running | 派生：`plan_state='planned' AND 子树有 task ∈ {claimed,running,review}` |
| Completed | 派生：子树全部 task = done |
| （Ignore） | `scope_state='closed' AND permanent`（D-03） |

Task 八态照搬需求（D-04 新增 `cancelled`）；`Blocked` 不是列（D-16），是 `v_task_blocked` 视图。

## 2.4 事件目录（v1 封闭集）

```text
图:     graph.published            {version, added[], removed[], summary}
节点:   node.spec_updated          {diff}
任务:   task.claimed               {owner, lease_token, attempt_seq}
        task.heartbeat             {}                    ← 高频，只续租不投影
        task.submitted             {paths[]}
        task.validated             {verdict, validator, report}
        task.done                  {artifact_id}
        task.failed                {reason, failure}
        task.lease_expired         {stale_owner}         ← actor=system:reaper
        task.handover_reported     {handover}            ← D-05 ③如实上报
        task.cancelled             {by, reason}
范围:   package.scope_changed      {from, to, reason, until, permanent}
契约:   contract.published         {artifact_id, version, bump}
        contract.impact_marked     {affected[]}          ← D-08 major
```

约束：投影表的每一次 UPDATE 必须能指认产生它的事件（A-03）；`task.heartbeat` 例外——只更新 `lease_expires`，不写 event 表（写入量与价值不成比；心跳丢失的最坏后果就是租约过期，本身有事件）。

## 2.5 派生视图

```sql
-- Blocked 及其原因（D-16 + FR-12.4 可视化阻塞原因）
CREATE VIEW v_task_blocked AS
SELECT t.id,
       CASE WHEN anc.id IS NOT NULL THEN 'scope:' || anc.scope_state
            ELSE 'deps:' || array_to_string(unmet.ids, ',') END AS reason
FROM node t
LEFT JOIN LATERAL (        -- 最近的受限祖先（物化路径前缀匹配，免递归）
    SELECT p.id, p.scope_state FROM node p
    WHERE p.kind='package' AND p.scope_state != 'active'
      AND t.path LIKE p.path || '.%'
    ORDER BY length(p.path) DESC LIMIT 1
) anc ON true
LEFT JOIN LATERAL (        -- 未满足依赖
    SELECT array_agg(e.from_id) AS ids FROM edge e
    JOIN node up ON up.id = e.from_id
    WHERE e.to_id = t.id AND (up.kind='task' AND up.task_state != 'done'
                              OR up.kind='package')
) unmet ON true
WHERE t.kind='task' AND t.task_state IN ('todo','ready')
  AND (anc.id IS NOT NULL OR unmet.ids IS NOT NULL);

-- Package 聚合进度（FR-12.2）
CREATE VIEW v_package_progress AS
SELECT p.id,
       count(*) FILTER (WHERE t.task_state='done')                          AS done,
       count(*) FILTER (WHERE t.task_state IN ('claimed','running','review')) AS active,
       count(*)                                                              AS total
FROM node p JOIN node t
  ON t.kind='task' AND t.path LIKE p.path || '.%'
WHERE p.kind='package' GROUP BY p.id;
```

`path` 物化路径让祖先/后代查询变成前缀 LIKE（走 `text_pattern_ops` 索引），递归 CTE 只在投影重建等离线场景使用。**path 由 parent_id 派生**，节点移动时同事务重写子树 path（v1 图演化只增不移，成本可忽略）。

## 2.6 claim-next（调度器本体，A-01）

```sql
WITH candidate AS (
    SELECT n.id FROM node n
    WHERE n.kind = 'task'
      AND n.ready                                  -- 预计算列：见下
      AND n.required_caps <@ :agent_caps
      AND NOT EXISTS (                             -- A-02 认领期写冲突防线
          SELECT 1 FROM node r
          WHERE r.kind='task' AND r.task_state IN ('claimed','running','review')
            AND r.write_scope && n.write_scope     -- 前缀相交（v1 收窄后可用数组重叠近似 + core 层精判）
      )
    ORDER BY n.priority DESC, n.id
    FOR UPDATE OF n SKIP LOCKED
    LIMIT 1
)
UPDATE node t SET
    task_state = 'claimed', owner = :actor,
    lease_token = gen_random_uuid(),
    lease_expires = now() + make_interval(secs => :ttl),
    updated_at = now()
FROM candidate WHERE t.id = candidate.id
RETURNING t.*;
```

**`ready` 列的维护契约**：`ready = task_state='todo' AND 全部上游 done AND 无受限祖先`。三类事务触达它——上游 task.done（重算直接下游）、package.scope_changed（重算子树）、graph.published（重算新增节点）。每类都在同一事务内完成投影更新；`dag-cli admin verify` 全量重算比对，漂移即 bug（A-04 双实现校验）。

> 写冲突判定的 SQL 近似：`write_scope && write_scope`（数组精确重叠）判不出前缀包含（`src/api` vs `src/api/user`）。v1 方案：claim 事务内取回候选后由 dag-core 的前缀函数精判，不符则跳过该候选重试——SKIP LOCKED 语义下代价是一次额外循环，正确性优先。
