## M1 · 图引擎与编译校验（D-01 的质量门，最高优先级）

**产出**：sunmao-core 完整校验器 + sunmao-store 图读写 + `POST /graph/publish` 端到端。

- [x] 环检测、悬空依赖、重复 ID、类型不变量、写范围前缀相交、缺失输入、Validator 未注册——FR-03.3 全清单
- [x] 违规聚合报告（一次返回全部，FR-03.4）
- [x] graph_version 乐观锁（D-12）
- [x] `ready` 预计算列的三类维护事务 + 全量重算校验函数（A-04 双实现一致性 property test）
- [x] path 物化 + 前缀查询

**验收**：构造 20 个非法图 fixture（每规则 ≥2）全部 422 且指认正确；合法图发布后 `ready_now` 与手算一致。
**为什么最先**：D-01 全自动发布后校验是唯一质量门（README 偏差表），它不完备整个系统就在裸奔。

## M2 · 调度与租约（系统心脏）

- [x] claim-next SQL + core 层前缀精判循环（02 §2.6 注）

## M3 · 完工链与接力（价值闭环）

**产出**：submit → diff 核验 → cmd Validator → commit → Done；接力 handover 全流程。

- [x] GitWorkspace：`status --porcelain` 解析、scope 分拣（邻居在制品忽略规则）、按范围 add+commit、checkout 丢弃
- [x] 内置 Validator：scope-diff / cmd / artifact-exists；注册表（FR-08.1，07 §7.5 注册式扩展）
- [x] 失败注入下次输入（D-06）；commit-后-PG-前的幂等重放（流程一注）
- [x] handover 块组装（在制品扫描 + 进度快照）；handover-review 前置约束（409）
- [x] Drain 与 `--force` 取消（D-04，含 Cancelled 态）

**验收**：端到端脚本——建 3 任务链（A→B→C）于真实 git 仓库，模拟 Worker 依序完成，断言 3 个 commit、artifact 记录、C 在 B done 前不可 claim；模拟越界写 → 422 且 reopened；模拟租约过期接力 → handover 块含在制品清单。

## M4 · Contract 语义与事件面（协作完整）

**产出**：contract publish/approve-major/影响标记；SSE；审计查询；CLI 首版。

- [x] semver 声明式 bump + major 挂起 + 人工放行 + 传递闭包影响标记（D-08，流程四 b）
- [x] SSE `/events/stream`（LISTEN → fan-out → after_seq 补拉，按项目过滤）
- [x] `GET /events` 分页；`admin/rebuild-projection`（A-03 重放）
- [x] sunmao 首版（D-19 交互模型）：
  - `sunmao config`——TUI 设置 base_url（ratatui 表单）→ `~/.config/sunmao/config.toml`
  - `sunmao projects`——TUI 项目列表/详情
  - `sunmao init`——注册项目 + 写 `.sunmao.toml` 目录绑定
  - 目录绑定解析链（--project > .sunmao.toml > git root 反查 > TUI 选择器）
  - `sunmao tree / task show / scope / approve-major / admin verify`（默认项目下）

**验收**：major 发布 → agent 调 approve 得 403、人调后下游节点带 needs_replan 标记；杀 server 重启，SSE 客户端 after_seq 续传无丢失；rebuild 后投影与增量维护结果 byte-level 一致；两个项目并存时事件流/claim 互不可见（D-19 隔离）；在项目目录外运行 `sunmao tree` 落入 TUI 选择器、目录内自动解析。

## 风险与提前量

| 风险 | 缓解 |
|------|------|
| sqlx 编译期检查需活 PG，CI 复杂化 | 用 `sqlx prepare` 离线模式，`.sqlx/` 入库 |
| git 子进程在大仓库慢（status 全树扫描） | v1 接受；预留 `--pathspec` 限定扫描范围的优化位 |
| LLM Planner 产出不合 publish 契约 | M1 的 422 报告设计成「可直接喂回 LLM 修正」的自然语言（FR-03.4 加严：报告面向 LLM 消费者） |
| 接力的在制品审查提示词不可靠 | handover-review 强制前置（409）兜底；审查质量差只影响效率不影响正确性 |
| 单人开发节奏 | 每个 M 都是可停顿点，M3 后系统已可日常自用 |
