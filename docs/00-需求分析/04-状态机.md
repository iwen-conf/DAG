# 04 · 状态机

> 原则：**Package 与 Task 使用两套状态机，禁止混用。**  
> Package 是管理对象；Task 是执行对象。

## 4.1 Package 生命周期（项目管理）

```text
Draft
  │
Planning
  │
Ready
  │
Running
 ├──────► Paused
 │            │
 │            ▼
 ├──────► Closed
 │
 ▼
Completed
  │
  ▼
Archived
```

| 状态 | 含义 |
|------|------|
| Draft | 刚创建 |
| Planning | 正在拆分 / 展开 |
| Ready | 可展开或子任务可进入执行准备 |
| Running | 子任务正在开发 |
| Paused | 人工暂停 |
| Closed | 人工关闭（暂不做） |
| Completed | 全部完成 |
| Archived | 历史归档 |

### Closed 语义（必须明确）

`Closed` 表示：

- 不能 Claim 其子 Task；
- 不能 Expand；
- 不能自动生成子任务；
- 不能被 Planner 修改；
- **只有人** `Open / Reopen Package` 之后，Planner 才能继续工作。

可选元数据：

```yaml
status: Closed
reason: 业务暂缓
owner: Product Manager
until: 2026-10-01
```

Scheduler 对 Closed 子树 **直接跳过**。

### 建议增加：Ignore

| 状态 | 语义 |
|------|------|
| Closed | 以后可能还做 |
| Ignore | 永远不做（如「QQ 登录，老板说不要」） |

Ignore 后 Planner 也不再展开。

## 4.2 Task 生命周期（执行）

```text
Todo
  │
  ▼
Ready
  │
  ▼
Claimed
  │
  ▼
Running
  │
  ├────► Failed
  │
  ▼
Review
  │
  ├────► Failed        ← 验证不通过（FR-08.3/08.4）
  │
  ▼
Done
```

> Review 由 Validator 判定：通过 → Done；不通过 → Failed。原图缺少 Review → Failed 边，与 FR-08.3「Validator 通过后才能 Done」互补。Failed 之后走向（重试 / 修复 Task / 重规划）取决于 D-06 / D-07 的决策。

对话中还出现 **Blocked**（例如父 Package Closed 时，子 Task 全部变为 Blocked）。

> **架构落地注**（见 [01-架构设计](../01-架构设计/README.md)）：
> - 新增 **Cancelled** 终态（D-04 `--force` 终止，语义区别于 Failed——非任务自身失败）；
> - Package 八态拆为 `plan_state` + `scope_state` 两个存储列，Running/Completed 为派生聚合（偏差 #5，映射表见 [02-数据模型 §2.3](../01-架构设计/02-数据模型.md)）。

| 状态 | 含义 |
|------|------|
| Todo | 已创建，依赖未满足或未就绪 |
| Ready | 可被认领 |
| Claimed | 已被某 Agent 占用 |
| Running | 执行中 |
| Review | 等待 / 正在验收 |
| Done | 完成 |
| Failed | 失败 |
| Blocked | 被**范围**阻塞（祖先 Package Closed / Paused / Ignore）；普通依赖未满足用 Todo 表达，不占用 Blocked |

> **Blocked 的实现方式待定**：若逐个落库，Close/Reopen 需级联写整棵子树，且与调度端的祖先状态检查形成双真相源；建议作为**派生视图**（真相源 = 祖先 Package 状态）。见 [15-决策记录.md](./15-决策记录.md) D-16。

## 4.3 认领示例

```yaml
task:
  id: user.login.api
  state: Ready
  owner: null
```

Agent：

1. `GET /task/next` → `user.login.api`
2. `POST /claim` →

```yaml
owner: agent-13
state: Running   # Claimed / Running 二选一或并存，待定 → 见 15-决策记录 D-05（建议：保留两态，租约自 Claim 起算）
```

其他 Agent 再来：不给。

## 4.4 两套状态机的边界

| 对象 | 会不会「真正 Running」 |
|------|------------------------|
| Package `Server` | 否（聚合态） |
| Task `Generate Login API` | 是 |

`Server` 的 Running 表示「子树中有任务在跑」，不是它自己在写代码。
