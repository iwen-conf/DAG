# 04 · 状态机

> 原则：**Package 与 Task 使用两套状态机，禁止混用。**  
> Package 是管理对象；Task 是执行对象。

## 4.1 Package 生命周期（项目管理）

```text
Draft
  │
Planning
  │
Ready
  │
Running
 ├──────► Paused
 │            │
 │            ▼
 ├──────► Closed
 │
 ▼
Completed
  │
  ▼
Archived
```

| 状态 | 含义 |
|------|------|
| Draft | 刚创建 |
| Planning | 正在拆分 / 展开 |
| Ready | 可展开或子任务可进入执行准备 |
| Running | 子任务正在开发 |
| Paused | 人工暂停 |
| Closed | 人工关闭（暂不做） |
| Completed | 全部完成 |
| Archived | 历史归档 |

### Closed 语义（必须明确）

`Closed` 表示：

- 不能 Claim 其子 Task；
- 不能 Expand；
- 不能自动生成子任务；
- 不能被 Planner 修改；
- **只有人** `Open / Reopen Package` 之后，Planner 才能继续工作。

可选元数据：

```yaml
status: Closed
reason: 业务暂缓
owner: Product Manager
until: 2026-10-01
```

Scheduler 对 Closed 子树 **直接跳过**。

### 建议增加：Ignore

| 状态 | 语义 |
|------|------|
| Closed | 以后可能还做 |
| Ignore | 永远不做（如「QQ 登录，老板说不要」） |

Ignore 后 Planner 也不再展开。

## 4.2 Task 生命周期（执行）

```text
Todo
  │
  ▼
Ready
  │
  ▼
Claimed
  │
  ▼
Running
  │
  ├────► Failed
  │
  ▼
Review
  │
  ▼
Done
```

对话中还出现 **Blocked**（例如父 Package Closed 时，子 Task 全部变为 Blocked）。

| 状态 | 含义 |
|------|------|
| Todo | 已创建，依赖未满足或未就绪 |
| Ready | 可被认领 |
| Claimed | 已被某 Agent 占用 |
| Running | 执行中 |
| Review | 等待 / 正在验收 |
| Done | 完成 |
| Failed | 失败 |
| Blocked | 被范围或依赖阻塞（如父 Package Closed） |

## 4.3 认领示例

```yaml
task:
  id: user.login.api
  state: Ready
  owner: null
```

Agent：

1. `GET /task/next` → `user.login.api`
2. `POST /claim` →

```yaml
owner: agent-13
state: Running   # 或先 Claimed 再 Running，实现时可二选一但需统一
```

其他 Agent 再来：不给。

## 4.4 两套状态机的边界

| 对象 | 会不会「真正 Running」 |
|------|------------------------|
| Package `Server` | 否（聚合态） |
| Task `Generate Login API` | 是 |

`Server` 的 Running 表示「子树中有任务在跑」，不是它自己在写代码。
