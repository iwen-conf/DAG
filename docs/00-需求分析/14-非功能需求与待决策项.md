# 14 · 非功能需求与待决策项

## 14.1 非功能需求

### 一致性与并发

1. Claim、状态迁移、写集合占用和 Artifact 发布必须具备并发安全保证。
2. 相同请求重试不得造成重复认领、重复完成或重复发布。
3. 任意已发布 DAG 版本必须保持无环、引用完整和类型不变量成立。

### Fail Fast 与可观测性

1. 环、写冲突、缺失输入、非法迁移和验证失败必须立即暴露，不得用兜底逻辑隐藏。
2. 每个错误应包含节点、操作、输入版本、依赖路径和底层根因。
3. 关键路径必须有结构化日志、指标和可串联的追踪标识。

### 可恢复性

1. Planner、Worker、Validator 或 Task Service 崩溃后，不应破坏任务所有权和图一致性。
2. 恢复过程不得把仍合法执行的任务错误地重复派发。
3. Artifact 发布应避免出现状态为 Done 但产物不可读的半完成状态。

### 可扩展性

1. 新增业务 Context 不得要求修改 Scheduler 核心。
2. Planner、Worker、Validator 应能按能力水平扩展。
3. Artifact 类型、Validator 和执行器应支持注册式扩展。

### 性能

1. Ready 查询不应每次遍历完整 DAG，应使用增量索引或等价机制。
2. 影响分析和重规划应限制在相关子图。
3. UI 应按层级懒加载，避免一次渲染全部节点。

### 安全与权限

> **v1 降级为非目标**（D-18 已确认：单人本地使用，无认证无 token）。以下条款保留作为多人/联网场景的升级需求：

1. 人工范围控制、规划发布、任务执行和 Artifact 读取应有独立权限。
2. Worker 应遵守最小权限，只能访问任务声明的输入与写范围。（写范围校验保留——由 D-10 的 diff 核验实现，属正确性而非安全）
3. 高风险操作及权限拒绝必须进入审计日志。（审计保留——由 D-17 事件表实现）

## 14.3 建议的首版范围（待用户确认）

首版建议聚焦：单项目、Package/Task 两类节点、人工批准规划、层级展开、Pull + 原子 Claim、基础能力标签、文件型 Artifact、命令型 Validator、人工 Close/Reopen、环和写冲突静态校验。

多 Planner、自动 Merge、复杂公平调度、跨项目资源治理、自动破坏性 Contract 升级可后置。
