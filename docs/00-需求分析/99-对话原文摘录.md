## 第 1 轮 · 如何拆分任务边界 / 上下文

- **原则**：按变化原因（Reason to Change）拆，不按数据拆  
- **七层**：业务动作 → 规则/Context → 共变数据 → 生命周期 → 事务边界 → Actor → Bounded Context  
- **五问**：谁 / 为什么 / 何时改 / 与谁同事务 / 将来谁改  
- **反模式**：8000 行 UserService  
- **映射**：Agent 按 Task Context 分，不按 Tool 堆

→ [08-边界拆分方法论.md](./08-边界拆分方法论.md)

## 第 3 轮 · 边界与并行

- 并行本质：无共享可变状态  
- Task 结构：Input / Output / Constraint / Owner  
- 下一代：IO 而非巨型 Context；边是 Artifact  
- 决策权：唯一写权限  
- **五条件**可并行；设计重心 = DAG + Contract
