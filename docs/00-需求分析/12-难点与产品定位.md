# 12 · 难点与产品定位

> 来源：完整对话第 5 轮及全文收束  
> 用途：明确「难在哪里」与「我们在做什么样的产品」。

## 12.1 DAG 不是难点

拓扑排序、环检测、并行调度、渲染图 —— 成熟且简单。

> 答案不是「没人想到 DAG」，而是**真正困难的地方根本不是 DAG**。

## 12.2 五大真正难点

### ① Task 是怎么来的？（Planning）

「开发一个小说网站」如何变成 Design Schema → OpenAPI → Go → Frontend → Tests？

这是**规划**，不是数据结构算法：

```text
自然语言 → Planner → DAG
```

### ② Task 的边界

`生成 User` / `生成 Novel` 可并行；但 `修改 User` / `Login` / `JWT` 的归属没有唯一标准答案 → Planner **必须懂架构**（结合 [08](./08-边界拆分方法论.md)）。

### ③ 上下文污染与循环依赖

跨模块读取很快形成环；Planner 必须拆环、引入 Port/事件/反腐层等策略。

### ④ Merge

两 Agent 改同一文件：Git 三方合并都难，AI 更难。  
应用层手段：唯一写权限、文件/目录所有权、冲突检测拒绝非法图。

### ⑤ 验证

Schema / OpenAPI / 代码不一致时，必须有 **Validator** 负责判定，不能靠「再聊一轮」。

## 12.3 现状：有人做，但 DAG 不是第一公民

| 产品/框架 | 对话中的评价 |
|-----------|----------------|
| Claude Code | 偏单 Agent 自规划，未把项目公开成 DAG |
| Codex | 规划与执行合一，用户不操作任务图 |
| Cursor | IDE + AI，管编辑流程而非项目任务图 |
| OpenHands / AutoGen / CrewAI / LangGraph | 有编排，但是 **Agent 工作流**，不是 **软件工程 DAG** |

共同特点：

> **DAG 是内部实现，而不是第一公民。**

## 12.4 反转：不要「会话」，只要 Artifact

```text
schema.sql → openapi.yaml → go/* → tests/*
```

Agent 无需知道整个项目，只需：

> 输入是什么？输出是什么？完成。退出。下次重新创建。

对比：

| 常见 Agent | Build System |
|------------|--------------|
| Memory 越来越长越慢 | Task → Input → Output → 结束 |
| 会话状态 | 无 Memory，只有 Artifact |

## 12.5 产品隐喻：AI Cargo / AI Bazel

Cargo：

```text
crate → dependency → build DAG
```

迁移到 AI：

```text
requirement.ai
    → planner
    → task graph
    → scheduler
    → artifacts
    → validator
```

所有 Agent 只是 **Worker**。

若体系做完整：

> 定位不是「另一个 Agent 框架」，而更像 **AI 时代的 Bazel/Cargo** ——  
> 负责规划、调度、验证和增量重建；底层大模型只是其中一种执行器。

并与项目协作结合：

> **GitHub Projects / Jira + Kubernetes 调度器** 的 AI 原生版本。

即 **AI Project Operating System**。

## 12.6 两类 DAG 不要混淆

| 类型 | 节点语义 | 典型系统 |
|------|----------|----------|
| 推理 DAG | LLM / Tool 步骤 | LangGraph 等 |
| **工程 DAG** | Package / Module / Feature / Task / Artifact | 本项目 |

本项目只把工程 DAG 作为第一公民。

## 12.7 成功标准（产品级）

1. 用户能看到并操作一张**可演化的**项目任务图（非黑盒）。  
2. 多 Worker 并行且**不抢任务、不双写**。  
3. 契约冻结后，跨端 / 跨层可安全并行。  
4. 失败只重规划子图，不全量推倒。  
5. 人可关闭/暂停 Package，AI 不能擅自改范围。  
6. 核心状态在 Task Service + Artifact Store，不在聊天窗口。

## 12.8 非目标（明确不做）

- 以长对话 Memory 作为项目真相源  
- Boss Agent 强推任务给 Worker  
- 一次性生成并冻结完整静态 DAG 后永不修改  
- 让 Scheduler 理解业务语义
